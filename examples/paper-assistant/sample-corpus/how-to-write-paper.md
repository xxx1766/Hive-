# 论文写作方法论（精简版 — 给 Agent 团队读）

## 三阶段

### Pre-Writing
- 文献综述 + 复现，圈定 gap
- 跑 motivation 实验（用真数据强化论证）
- 写假设、目标、初稿标题；准备目标会议/期刊的模板
- 综述 ~30 篇，按"逐步发展的故事"组织
- 列章节大纲 + 页数预算

### Active Writing
- **图先**：占位框（白框）也要先排好
- Methods（突出核心 idea）
- Results / Analysis（按 outline 重排，不按实验时间）
- Related Work（裁剪到 ~30 条）
- Introduction（最后写，宏观→具体）
- Abstract / Title / Keywords

### Post-Writing
- 归档数据 / 代码 / 图脚本
- 实验记录 / 失败 trial 文档化

## 痛点 / 反模式

- "一定要动手跑一跑代码" —— 别在脑里推
- "Introduction 跟后面的每一章节都有关联，真'动全身而牵一发'" —— 留到最后
- 慎用 "novel / first time / paradigm-changing" —— "会被狠狠 challenge 的"
- 初稿不追求完美 —— 浪费精力
- 不要试图引用所有相关工作 —— 选择性 > 完整性

## 启发式

- 写作顺序：figures → methods → results → discussion → related → intro → abstract → title
- 每图 3–4 个数据组，单页一图，灰度可读
- 时间分配：零碎时间写容易，大块时间（3–5 天）写难
- 一定要专注：3–5 天 deep work 啥也不干
